Bibliographic Metadata

Publication ➞ Bibliographic Resources

The use of Typed Links in the HTTP Link header is illustrated for the paper:

Lotts, M. Building bridges, creating partnerships, and elevating the Arts: The Rutgers University Art Library Exhibition Spaces. College & Research Libraries News 77, 5 (May 2016), 226–230. http://crln.acrl.org/content/77/5/226.full.:

This is a summary of the links that should be provided:

  • The Entry Page, which is an HTML rendering of the article, should provide a link with the describedby relation type pointing to each associated Bibliographic Resource. Each link should use the type attribute (and the profile attribute, if needed) to specify the metadata format.
  • Each Bibliographic Resource provided by the portal should provide a link with the describes relation type pointing back to the Entry Page.

Link Headers for the Bibliographic Metadata pattern




The HTTP Link Header to link to metadata from the Publication HTML URI
http://crln.acrl.org/content/77/5/226.full is:

curl -I "http://crln.acrl.org/content/77/5/226.full"

Link: <http://crln.acrl.org/citmgr?type=refman&gca=crlnews%3B77%2F5%2F226>
      ; rel="describedby"
      ; type="application/x-research-info-systems",
      <http://crln.acrl.org/citmgr?type=bibtex&gca=crlnews%3B77%2F5%2F226>
      ; rel="describedby"
      ; type="application/x-bibtex"

The HTTP Link header to link from the Metadata RIS URI
http://crln.acrl.org/citmgr?type=refman&gca=crlnews%3B77%2F5%2F226 is:


curl -I "http://crln.acrl.org/citmgr?type=refman&gca=crlnews%3B77%2F5%2F226"

Link: <http://crln.acrl.org/content/77/5/226.full>
      ; rel="describes"

The HTTP Link header to link from the Metadata BibTeX URI
http://crln.acrl.org/citmgr?type=bibtex&gca=crlnews%3B77%2F5%2F226 is:


curl -I "http://crln.acrl.org/citmgr?type=bibtex&gca=crlnews%3B77%2F5%2F226"

Link: <http://crln.acrl.org/content/77/5/226.full>
      ; rel="describes"

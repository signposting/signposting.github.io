The use of Typed Links in the HTTP Link header is illustrated for the paper:

Baxter, H. A. Framework for Remediation of Rivers Impacted by Legacy Metal Mine Pollution. PhD thesis, The University of Hull, Dec. 2015. https://hydra.hull.ac.uk/resources/hull:13221.

This is a summary of the links that should be provided:

  • The landing page should provide a link with the describedby relation type pointing to each associated Bibliographic Resource. Each link should use the type attribute (and the profile attribute, if needed) to specify the metadata format.
  • Each Bibliographic Resource provided by the portal should provide a link with the describes relation type pointing back to the landing page.

Link Headers for the Bibliographic Metadata pattern




The HTTP Link Header to link to metadata from the Landing Page URI
http://eprints.soton.ac.uk/338797 is:

curl -I "https://hydra.hull.ac.uk/resources/hull:13221"

Link: <https://hydra.hull.ac.uk/assets/hull:13221?datastream_id=descMetadata>
      ; rel="describedby"
      ; type="application/xml"
      ; profile="http://www.loc.gov/mods",
      <https://hydra.hull.ac.uk/resources/hull:13221.oai_dc_xml>
      ; rel="describedby"
      ; type="application/xml"
      ; profile="http://purl.org/dc/elements/1.1/"

The HTTP Link header to link from the Metadata MODS URI
https://hydra.hull.ac.uk/assets/hull:13221?datastream_id=descMetadata is:


curl -I "https://hydra.hull.ac.uk/assets/hull:13221?datastream_id=descMetadata"

Link: <https://hydra.hull.ac.uk/resources/hull:13221>
      ; rel="describes"
